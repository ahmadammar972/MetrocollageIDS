6 Phases Cyclic Structure Iterative Process
Metro College of Technology
Data Analytics Lifecycle
Introduction
Core Concept

What is the Data Analytics Lifecycle?

As data gets created, consumed, tested, processed, and reused, it goes through several phases during its entire life.

A data analytics architecture maps out these steps for data science professionals. It is a cyclic structure that encompasses all lifecycle phases — each stage with its own significance and characteristics.

🔄 Why Cyclic?

The circular form guides professionals to move forward or backward through stages. Based on new information, you may scrap research and return to the first step to redo the entire analysis.

Key Characteristics

♻️
Iterative
Can move forward or backward at any point
🧩
Flexible
Phases can be added, removed, or run in parallel
🎯
Goal-Driven
Every phase is anchored to the initial business objective
Metro College of Technology
Data Analytics Lifecycle
Flexibility
Important Note

Flexibility of the Lifecycle

No single universal structure exists. While experts discuss the lifecycle, there is still no uniformly agreed-upon architecture that every professional follows.

➕ Add phases when the problem complexity demands extra steps beyond the basics
➖ Remove phases when a simpler analysis doesn't require every step
⚡ Parallel work — it's possible to work on multiple stages simultaneously
⏭️ Skip a phase entirely if it's not applicable to a specific project context

Why These Phases?

The phases listed in this lecture represent the fundamentals of almost every data analysis process. Even in the most flexible implementations, these core activities will be present in some form.

💡 Think of these phases as a baseline template — always adaptable to the needs of the specific project and organisation.

The 6-Phase Lifecycle

2 · Data Prep
3 · Model Plan

Phase 1 — Discovery & Formation

Define the data's purpose, map out requirements, and formulate Initial Hypotheses (IHs). Focus on enterprise needs, not the data itself. Assess tools needed for processing.

🎯 Everything downstream is anchored to the goal set here.

Phase 2 — Data Preparation & Processing

Collect, process, and cleanse data. Ensure needed data is available. Collection methods: Data Acquisition (external), Data Entry (digital/manual), Signal Reception (IoT devices).

📊 Attention shifts from business requirements to information requirements.

Phase 3 — Model Planning (Design)

Build the model plan using ETL, ELT, or ETLT techniques. Identify relationships between data points, select key variables, and plan the model structure collaboratively.

🛠️ The blueprint for everything in Phase 4.

Phase 4 — Model Building

Develop datasets for testing, training, and production. Execute the model using decision trees, regression, and neural networks. Run trial tests to verify output matches expectations.

🔬 Where theory becomes working reality.

Phase 5 — Result Communication & Publication

Collaborate with stakeholders to evaluate success/failure. Identify key findings, measure business value, and produce a narrative that conveys results clearly to non-technical audiences.

📣 Story matters — data must be communicated, not just computed.

Phase 6 — Measuring Effectiveness

Provide detailed reports, technical documents to stakeholders. Move data to a live environment and monitor results. If outcomes deviate from Phase 1 goals, loop back and revise.

📈 Closes the loop — or sends you back to refine it.

Phase 3 of 6
📐 Phase 3 · Model Planning

Phase 3 — Design a Model

With business goals mapped and data collected, it's time to build a model that utilises the data to achieve those goals. This phase is about planning — not yet executing.

Data Loading Techniques

ETL — Extract, Transform, Load

Transforms data first using business rules, then loads it into the sandbox. Ideal when data quality rules are well-defined upfront.

ELT — Extract, Load, Transform

Loads raw data into the sandbox first, then transforms. Better for large-scale cloud environments.

ETLT — Extract, Transform, Load, Transform

A hybrid with two transformation levels — initial cleaning before load, deeper transformation after.

Data Driven Decision Making
Definition
🎯 What is DDDM?

What is Data Driven Decision Making?

Data Driven Decision Making (DDDM) is the practice of basing strategic and operational decisions on analysed data and measurable evidence rather than intuition, experience alone, or guesswork.

📌 It starts with a question

Every DDDM cycle begins with a clearly defined business question or problem that requires a decision.

📊 Data is the input

Relevant, high-quality data is gathered from internal and external sources and analysed to extract insights.

✅ Action is the output

Insights translate directly into decisions, strategies, or recommendations that can be measured and reviewed.

Intuition vs. Data-Driven

Data Driven Decision Making
The Process
🔄 DDDM Steps

The DDDM Process

Data driven decisions follow a repeatable cycle — each step builds directly on the last.

❓
Step 1 — Define the Question
Identify the specific decision to be made. A vague problem produces vague data. Be precise: "Which product category drives most churn?"
🗂️
Step 2 — Collect Relevant Data
Gather internal (CRM, ERP, sales logs) and external (market reports, surveys, social media) data aligned to the question.
🧹
Step 3 — Clean & Prepare Data
Remove duplicates, handle missing values, standardise formats. Poor data quality leads to poor decisions regardless of how good the analysis is.
📊
Step 4 — Analyse Data
Apply descriptive, diagnostic, predictive, or prescriptive analytics depending on the type of decision needed.
💡
Step 5 — Interpret & Decide
Translate findings into a clear recommendation. Communicate using visualisations and narratives tailored to stakeholders.
📈
Step 6 — Monitor & Refine
Track outcomes against the original goal. If results deviate, loop back with new data to refine the decision.

Data Driven Decision Making
Why It Matters
🚀 Benefits

Why Use Data Driven Decision Making?

🎯

Improved Accuracy

Decisions backed by data are more likely to reflect reality. Reduces the impact of cognitive biases and personal preferences on outcomes.

Example: Netflix uses viewing data to decide which shows to renew — not executive instinct.
⚡

Faster Decisions

Pre-built dashboards and real-time analytics allow managers to act quickly without waiting for manual reports. Competitive edge in fast-moving markets.

Example: Amazon's pricing algorithm adjusts millions of prices daily using real-time demand signals.
🔁

Continuous Improvement

DDDM creates a feedback loop. Each decision generates new data, which improves the quality of future decisions over time.

Example: Spotify's recommendation engine improves with every song skip, play, or playlist save.

⚠️ Barriers

Barriers to Data Driven Decision Making

Despite its advantages, many organisations struggle to fully adopt DDDM. Understanding barriers is the first step to overcoming them.

🗄️ Poor Data Quality

Incomplete, inconsistent, or outdated data leads to flawed analysis. "Garbage in, garbage out" — the quality of decisions is bounded by data quality.

🏢 Organisational Silos

Data trapped in separate departments prevents a unified view. Finance, sales, and operations may each hold key pieces that are never combined.

🙅 Resistance to Change

Employees and managers accustomed to intuition-based decisions may distrust or ignore data insights, especially when they challenge existing beliefs.

🔐 Privacy & Compliance

Regulations such as GDPR and PIPEDA restrict how personal data can be collected and used. Compliance requirements shape what data is even available for analysis.

🧩 Skills Gap

Many organisations lack staff trained in data literacy, analytics, or interpretation. A decision is only as good as the person reading the data.

💡 Key Insight: Organisations that invest in data literacy training across all levels — not just technical teams — see the highest ROI from DDDM initiatives.

🌍 Industry Examples

DDDM in Practice

🛒
Retail — Walmart
Inventory & Demand Forecasting

Walmart analyses over 1 million customer transactions per hour. Before Hurricane Frances in 2004, data showed beer and Pop-Tarts were top pre-storm sellers — stores pre-stocked accordingly, increasing sales significantly.

🏥
Healthcare — NHS
Patient Flow & Resource Planning

The UK's NHS uses patient admission data and predictive models to anticipate A&E demand by day and hour. This reduces waiting times and ensures staffing levels match real need.

🏦
Finance — JPMorgan Chase
Fraud Detection

Processes billions of transactions using machine learning models that flag anomalies in real time. Data-driven rules have reduced false positives while catching actual fraud patterns faster than manual review ever could.

⚽
Sports — FC Barcelona
Player Performance & Recruitment

Uses match data, GPS tracking, and biometric sensors to evaluate player performance and reduce injury risk. Transfer decisions are increasingly supported by analytics rather than scouts alone.

✈️
Aviation — Delta Airlines
Operations & Delay Reduction

Delta analyses weather, crew availability, gate utilisation, and maintenance logs simultaneously to proactively reroute flights and minimise delays — saving millions annually in operational costs.

📌 Common thread: In every case, organisations first defined a clear decision to make, then gathered the right data — not all data — to answer it.

🏗️ Building the Culture

Building a Data Driven Culture

DDDM is not just a technical initiative — it requires a shift in mindset, process, and leadership across the entire organisation.

1 · Leadership Buy-In
Executives must model data-driven behaviour. When leadership asks for data before making decisions, teams follow. "HiPPO effect" (Highest Paid Person's Opinion) is the enemy of DDDM.
2 · Data Literacy for All
Everyone — from analysts to HR to finance — should understand how to read a dashboard, interpret a trend, and question a data source. Literacy does not mean coding skills.
3 · Accessible Infrastructure
Data must be findable and usable by the right people at the right time. Data catalogues, self-service BI tools (Tableau, Power BI), and cloud platforms lower the barrier to entry.
4 · Incentivise Evidence
Reward teams that propose decisions with supporting data. Build data checkpoints into project approval and strategy review processes.

Data Maturity Levels

Level 1 — Data Aware: Data exists but is siloed and inconsistent. Decisions still largely intuition-based.
Level 2 — Data Informed: Some decisions use reports, but only retrospectively. Analytics is reactive.
Level 3 — Data Driven: Decisions are consistently backed by data. Predictive analytics informs strategy.
Level 4 — Data Native: Data is embedded in every process. AI and automation drive continuous optimisation.

DDDM Wrap-Up
🎯 Key Takeaways

Data Driven Decision Making — Summary

1️⃣
DDDM replaces guesswork with evidence
Every significant business decision benefits from data — even if that means confirming an intuition.
2️⃣
The process is a cycle, not a one-off
Define → Collect → Clean → Analyse → Decide → Monitor → Repeat. Each cycle improves the next.
3️⃣
Data quality is non-negotiable
"Garbage in, garbage out." Clean, relevant, timely data is the foundation of every reliable decision.
4️⃣
All 5 analytics types serve DDDM
Descriptive, Diagnostic, Predictive, Prescriptive, and Cognitive analytics each answer a different question in the decision-making chain.
5️⃣
Culture eats strategy for breakfast
Technical tools alone won't make an organisation data-driven. Leadership, literacy, and incentives must align.
